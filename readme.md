Некоторые вводные и допущения

1. За пределами рассмотрения остался вопрос дизайна источника данных и подключения к нему. Предполагается, что взаимодействие с условным репозиторием 
будет вестись через интерфейс ICurrencyRateStorage с помощью какого-нибудь DataPrividerLocator с использованием моста между функциональными моделями 
и моделью хранения данных
2. Некоторые элементы конструкции явно избыточны, такие как ICurrency - они использованы для демонстрации способности конструкции функционального модуля 
к расширению без изменений
3. Единственная функция Web Api сделана чиста ради второго варианта тестирования и демонстратора использования (с учетом п.1 - там тоже вместо данных заглушка)
