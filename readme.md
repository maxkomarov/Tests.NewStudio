Некоторые вводные и допущения

1. Предполагается простая структура хранения данных о курсах, а именно
	- Исходная валюта
	- Целевая валюта
	- Провайдер курса
	- Дата/время установки курса

	Идентификаторами валют служат их буквенные коды согласно ISO4217 
		Очевидно, это не лучший выбор, так как ухудшается качество поиска (числовой код был бы предпочтительнее), 
		однако это улучшает качество представления без введения специального справоника валют
	Идентификатором провайдера курса (т.е. банка) служит некий условный код, который может представлять собой IBAN, SWIFT-BIC или просто ИНН/МФО
		В силу определенного упрощения задачи специальный справочник для этого не вводим, но при нужде система расширяется, 
		если для хранения связи курс-провайдер использовать его в качестве FK
	Дата/время полагается разумным хранить с точностью до часа
	Комбинация кодов валют, кода провайдера и даты/времени создают уникальный индекс курса, с помощью которых блокируется создание дубликатов 
	и можно оптимизировать поиск на уровне базы

2. Реализация задачи посредством принципа SR осуществляется сервисами
	Доступа к данным курсов в хранилище, обеспечивающего добавление, модификацию и удаление записей
	Поиска обменного курса по заданным критериям (из->в, на дату/время, лучший или конкретного провайдера)
	Калькулятора денежных сумм (однако, введенное доп.условие наличия провайдера курсов делает эту задачу достаточно широкой)


3. Проект строится по технологии TDD
	Тестируем наличие хранилища
	Тестируем добавление образцовых записей
	Тестируем поиск записей
	Тестируем сложение одиновалютных сумм
	Тестируем вычитаение одиновалютных сумм
	Тестируем сложение разновалютных сумм
	Тестируем вычитаение разноовалютных сумм
	Тестируем удаление записей
